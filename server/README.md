# FindAuto — Backend API

REST API для Telegram Mini App «FindAuto»: авторизация по Telegram `initData`,
хранение в SQLite, эндпоинты для поисков, избранного, уведомлений и профиля.

## Стек

- Node.js + Express + TypeScript
- SQLite (better-sqlite3)
- Валидация тела запросов через zod

## Запуск

```bash
cd server
npm install
cp .env.example .env      # при необходимости отредактируйте
npm run dev               # http://localhost:8000
```

Скрипты:

| Команда         | Действие                                  |
| --------------- | ----------------------------------------- |
| `npm run dev`   | Запуск с автоперезагрузкой (tsx watch)    |
| `npm run build` | Компиляция в `dist/`                       |
| `npm start`     | Запуск собранной версии                   |
| `npm run seed`  | Ручная загрузка каталога авто в БД        |

## Переменные окружения (`.env`)

| Переменная              | Назначение                                                            |
| ----------------------- | -------------------------------------------------------------------- |
| `PORT`                  | Порт API (по умолчанию 8000)                                          |
| `TELEGRAM_BOT_TOKEN`    | Токен бота из @BotFather. Нужен для валидации `initData` в проде.     |
| `ALLOW_INSECURE_AUTH`   | `true` — разрешить вход без `initData` (только для разработки).       |
| `INIT_DATA_MAX_AGE_SEC` | Максимальный возраст `initData` (защита от повторного использования). |
| `CORS_ORIGIN`           | Разрешённые Origin через запятую, или `*`.                           |
| `DATABASE_PATH`         | Путь к файлу SQLite.                                                  |

> В production обязательно задайте `TELEGRAM_BOT_TOKEN` и `ALLOW_INSECURE_AUTH=false`.

## Авторизация

Фронтенд шлёт заголовок:

```
Authorization: tma <initData>
```

Сервер проверяет подпись по алгоритму Telegram
(`secret_key = HMAC_SHA256(bot_token, "WebAppData")`), проверяет свежесть
и создаёт/обновляет пользователя в БД. Новому пользователю автоматически
заливается демо-контент (стартовые поиски, уведомления, избранное).

В режиме `ALLOW_INSECURE_AUTH=true` при отсутствии/невалидности `initData`
используется тестовый пользователь — чтобы фронтенд работал локально без бота.

## Парсинг площадок

Поиск запускается автоматически при **создании** поиска (фоново) и по кнопке
«Обновить». Логика — в `src/services/search-runner.ts`, адаптеры — в `src/parsers/sources/`.

| Площадка | Способ | Статус |
| --- | --- | --- |
| **Onliner** | Открытый JSON API `ab.onliner.by/sdapi/ab.api` | ✅ работает из коробки |
| **Kufar** | Открытый JSON API `api.kufar.by/search-api` | ✅ работает из коробки |
| **av.by** | Headless-браузер (patchright), обход SafeLine WAF | ⚠️ опционально, см. ниже |

### av.by — обход SafeLine WAF

av.by закрыт антиботом **SafeLine** (интерактивный челлендж «Confirm You Are Human»
+ детект автоматизации через CDP `Runtime.enable` → «Debugging Detected»). Обычный
`fetch` и простые парсеры получают только страницу-челлендж (403/468).

Обход — реальный headless-браузер через **patchright** (патченный Playwright, скрывает
утечку `Runtime.enable`). Настройка **на сервере (Ubuntu 22.04)**:

```bash
cd server
npm install
npx patchright install --with-deps chromium   # браузер + системные библиотеки

# 1) Проверить обход и разведать формат данных:
node verify-avby.mjs
#    Если "SafeLine passed: true" и печатаются ключи оффера — обход работает.
#    По этим ключам при необходимости поправьте маппер в src/parsers/sources/avby.ts

# 2) Включить источник:
#    в .env → AVBY_ENABLED=true
```

> **Важно:** на Windows патченные форки Playwright не запускаются (`spawn UNKNOWN`) —
> это ограничение ОС, не кода. Проверять и запускать av.by нужно на Linux-сервере.
> Пока `AVBY_ENABLED=false`, источник тихо пропускается и не влияет на Onliner/Kufar.
> SafeLine — движущаяся цель: если обход перестанет работать, альтернатива —
> платный scraping-API (ScrapFly/ZenRows) в том же адаптере.

## Уведомления в личку Telegram

Бот пишет пользователю в личные сообщения ([src/telegram/](src/telegram/)):

- **Через 30 секунд после создания поиска** — сводка: «По вашему запросу
  {марка + все фильтры} найдено N объявлений».
- Затем, **если поиск активен** — по отдельному сообщению на каждое авто:
  «Мы нашли новую машину по вашим характеристикам …» + ссылка на объявление
  (не более 10 за раз, с паузами — защита от лимитов Telegram).
- При повторных проверках (кнопка «Обновить», в будущем — cron) сообщения
  приходят только по **новым** объявлениям активного поиска.

Требования:

- Задайте `TELEGRAM_BOT_TOKEN` в `.env` (тот же токен из @BotFather, что и для
  валидации initData).
- Пользователь должен был **хотя бы раз запустить бота** (нажать Start / открыть
  Mini App через бота) — иначе Telegram запрещает боту писать первым (ошибка 403),
  сообщение просто не отправится (залогируется, приложение не падает).
- Без токена рассылка тихо отключена; Onliner/Kufar и сам поиск работают.
